---
name: explain-code
description: Explains code with visual diagrams and analogies. Use when explaining how code works, teaching about a codebase, or when the user asks "how does this work?"
---

When explaining code, always include:

1. **Draw a diagram**: Use ASCII art to show the flow, structure, or relationships
2. **Walk through the code**: Explain step-by-step what happens. walk thqough the code as a complete flow, not just line-by-line. Show how data moves and changes. explain what happens in each function and how they connect.

Keep explanations conversational. For complex concepts, use multiple analogies.
